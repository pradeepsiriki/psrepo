what is SDLC:
=============

on a high level, we have 3 things in developmet process

1. Design

2. Develop

3.  Testing

Let's say we are working at ps96.com , we are building an e-commerce app, the below is the cyclic process

1. Planning+requirements => gets the requirement from customers (if applicable)
2. Defining => Mostly paperwork
3. Designing => Here they write High level design and Low level design, scaling and modules, s/ws , tech stack will come in action
Here comes the Devops and Automation scope, The below brings the automation phase onto the table

4. Building => Developing (putting in Git or any SCM) and packaging
5. Testing => Testing the s/w , unit , functional, regression, integration testing come here
6. Deploy => Make the s/w available for the customer

DevOps Engineer should make sure the 4,5,6 steps are automated to improve the efficiency 

only goal is to how do we improve the org's efficiency in attractig and delivery to the users 

Here comes the sdlc methods like Agile comes in action!
